Six approaches (visibly win on)

  • • vanilla-rag — baseline (dense top-k)
  • • hybrid-rag — exact keyword / ID queries
  • • contextual-rag — context-starved chunks (Docling)
  • • graph-rag — thematic / whole-corpus
  • • agentic-rag — multi-hop / comparative
  • • n8n-adaptive-rag — mixed simple+complex

Model routing

  • • LiteLLM routes to the active Atlas provider source
  • • Generation model choices live in roles.yaml
  • • Request properties stay scoped per model via models.yaml
  • • LightRAG role models use Atlas LIGHTRAG_* inputs
  • • Embeddings default to nomic-embed-text (768-dim)
