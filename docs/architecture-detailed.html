RAG Showcase - Detailed Architecture

Atlas stack + plugin seam + six RAG endpoints + local Apple Silicon model routing

1. Request surface

  • OpenWebUI and the comparison harness both call the same LiteLLM models.
  • Every approach returns a uniform answer, source, and metrics payload.
  • The judge panel is offline and reads stored matrix outputs.

2. Plugin boundary

  • Atlas owns the infrastructure; rag-showcase adds a mounted FastAPI plugin.
  • register_models.py makes each approach appear as a LiteLLM model.
  • roles.yaml and models.yaml keep model choice and request properties scoped.

3. Local model strategy

  • macOS containers cannot use Metal GPU acceleration, so large LLM calls go to host Ollama.
  • Generation uses Qwen with think:false; LightRAG uses a non-reasoning model.
  • Embeddings, extraction, query generation, and judging stay local.
