# Corpus

## 1. Contents

`corpus/` provides two showcase corpora:

- `raw/` is generated by `fetch_corpus.py` from a MultiHop-RAG subset plus the
  hand-picked keyword docs in `keyword_docs/` (rare identifiers like
  `WIDGET-ERR-7741` that exercise the exact-keyword contrast). The populated
  `corpus/raw/` directory is gitignored; `fetch_corpus.py` is the source of truth.
- `graph_native/` is committed source data: 10 concise, relation-dense real-world
  dossiers with explicit `Relations:` bullets. It is designed to test whether
  `graph-rag` benefits from entity bridges, shared actors, timelines, and
  relationship-chain prompts.

## 2. Setup

```bash
python3 corpus/fetch_corpus.py
```

This copies the keyword docs into `corpus/raw/` and appends a MultiHop-RAG slice.
The MultiHop-RAG slice needs the optional `datasets` package, which is kept out
of the uv-managed deps (its transitive closure is heavy) — install it on demand:

```bash
python3 -m pip install datasets
```

If `datasets` is unavailable, the script degrades gracefully to keyword-docs only.

The graph-native corpus is already committed and does not need a fetch step. Use
[`demo/graph_native_queries.yaml`](../demo/graph_native_queries.yaml) with the
comparison harness to run that evaluation set.
